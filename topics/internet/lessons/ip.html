IP & Routing

Once you know the destination's IP address, your packet has to get there — hopping from one router to the next across an ocean of networks that have never met. No central map. No single owner. Just routers, gossiping.

The model

Every machine on the internet has an IP address. Routers sit at the edges between networks (your home → ISP → backbone → cloud provider → server). Each router has a routing table that maps "if the destination matches this subnet, send to this neighbor."

  • Packets are independent. Each one gets routed afresh; two packets from the same flow can take different paths.
  • Each packet has a TTL (Time To Live) — a hop counter that decrements at every router. If it hits zero, the packet is dropped. This prevents routing loops.
  • The path you actually take is the result of thousands of independent routing decisions. traceroute walks it.

Try it

Send a packet across a small network. Watch TTL tick down at each hop.

Key takeaways

  • IP is best-effort and connectionless. Each packet is independent. Reliability is TCP's job, one layer up.
  • Routers don't know the full path — only "the next hop closer to the destination."
  • TTL prevents infinite loops if the routing table is misconfigured.
  • BGP (Border Gateway Protocol) is how the world's networks announce reachability to each other. It's basically gossip at internet scale.
