Hashing

A hash function takes any input and produces a fixed-size "fingerprint." It's one-way: easy to compute, infeasible to reverse. This one property is the foundation of nearly every other primitive in this topic.

The three properties

  • Deterministic — same input always produces the same hash.
  • One-way (preimage resistance) — given a hash, you can't find the input that produced it. The only way is to try every possible input.
  • Collision-resistant — you can't find two different inputs that produce the same hash. (Mathematically inevitable that they exist, but finding one is infeasible.)

Plus a useful side effect:

  • The avalanche effect — change one bit of input, roughly half the output bits flip. There's no "small input change = small output change" intuition.

Try it

Type anything. Watch the SHA-256 hash. Change one character — watch the entire fingerprint scramble.

Key takeaways

  • Hashes are not encryption — there's no key, and there's no way to recover the input. That's the whole point.
  • SHA-256 is the modern workhorse (256 bits = 64 hex chars). MD5 and SHA-1 are broken — don't use for security.
  • Common uses: integrity (was this file tampered with?), commitments (prove you knew X without revealing it), Merkle trees (Git, blockchains), HMAC, signatures.
  • Hashing user passwords directly is wrong. Lesson 7 covers what to do instead.
