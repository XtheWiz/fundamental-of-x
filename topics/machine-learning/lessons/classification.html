Lesson 04

Classification

Regression predicts a number; classification predicts a label. Spam or not. Cat or dog. The model's job is to draw a boundary in feature space — and the loss measures how confident it is on the right side.

From line to logistic

Linear regression outputs w·x + b, an unbounded number. For two-class problems we squash it through a sigmoid: σ(z) = 1 / (1 + e^−z), which maps any real number into (0, 1) — interpretable as a probability.

That's logistic regression: linear underneath, squashed on top. The decision boundary is where the probability crosses 0.5 — i.e., where w·x + b = 0. A line in 2D, a hyperplane in higher dimensions.

Why squared error doesn't work

For classification, MSE makes a bad loss — the gradient shrinks as predictions get confidently wrong, slowing learning where it should speed up. Cross-entropy (log loss) is the standard: heavily penalize confident wrong predictions, gently reward confident right ones.

Key takeaways

  • Logistic regression draws a straight decision boundary. Some problems (XOR, concentric rings) can't be solved by a line — that's what neural nets fix.
  • The output is a probability, not a label. The threshold (default 0.5) is a separate decision that depends on the cost of false positives vs false negatives.
  • Multi-class: replace sigmoid with softmax, replace binary cross-entropy with categorical cross-entropy. Same idea, more outputs.
  • Despite "regression" in the name, logistic regression is the classification baseline used in every ML library.
