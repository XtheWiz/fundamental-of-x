What is Learning?

Programming: you write a function that turns input into output. Machine learning: you write a shape for that function, and an algorithm picks the parameters that make it fit the data. The data does the work the programmer would have done.

The function-fitting view

Every supervised ML problem can be written as: given input x, produce output y. You guess the shape (linear, polynomial, neural network) and the algorithm finds the numbers (weights) that make the function agree with as many training examples as possible.

The three pieces of any ML system:

  • Model — the function shape, with adjustable parameters.
  • Loss function — a measure of how wrong the current predictions are.
  • Optimizer — an algorithm that nudges the parameters to lower the loss.

The three flavors

  • Supervised — you have labelled examples (x, y). The model learns to predict y from x. Classification, regression.
  • Unsupervised — only x, no labels. The model finds structure: clusters, dimensions, anomalies. Lesson 7 covers k-means.
  • Reinforcement — an agent acts in an environment, gets rewards. Learns a policy. Game-playing AIs, robotics.

Key takeaways

  • ML doesn't escape rules — it just finds them automatically by minimizing a loss.
  • Picking the loss function is a design decision. "Get this right" is too vague; "minimize squared error" or "maximize log-likelihood" is what an optimizer can act on.
  • Same algorithm, vastly different problems, just by swapping data and model shape.
  • Most "AI" today is supervised learning at scale. RL grabs headlines (AlphaGo, robot dogs), but it's a smaller slice of production.
