Gradient Descent

The optimizer that powers nearly all of modern ML. The intuition is one sentence: at every step, take a tiny step downhill on the loss surface. The hard part is "tiny" — too small and you crawl, too big and you bounce past the minimum.

The recipe

  1. Compute the loss at the current parameters.
  2. Compute the gradient — the direction of steepest increase in loss.
  3. Step in the opposite direction by learning_rate × gradient.
  4. Repeat until the loss stops shrinking.

The gradient is just calculus: for each parameter, how much does the loss change if you nudge that parameter? Stack those partial derivatives into a vector and you have the direction to move.

The learning rate

The most consequential hyperparameter:

  • Too small: convergence takes forever.
  • Too big: you overshoot the minimum, possibly diverge.
  • Just right: smooth descent.

Modern optimizers (Adam, RMSprop) adapt the rate per parameter, often reducing it over time. But the basic descent loop is still inside.

Try it

Roll a ball down a 1D loss curve. Tune the learning rate. Watch it converge, oscillate, or explode.

Key takeaways

  • Gradient descent works on any differentiable loss. That's why it generalizes from regression to deep nets without changing.
  • "Diverged with NaN loss" almost always means: learning rate too high.
  • Stochastic gradient descent (SGD) computes the gradient on a small batch instead of all data — much faster per step, slightly noisier path. The default in practice.
  • Non-convex losses (deep nets) have many local minima. Surprisingly, this rarely matters — SGD's noise + high-dimensional geometry usually find good ones.
