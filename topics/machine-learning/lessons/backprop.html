Backpropagation

How does the optimizer know which way to nudge weight #4,073 in the middle of a 200-layer network? Backprop: apply the chain rule, layer by layer, in reverse. Same calculus a physics student knows; just a lot of it.

The chain rule, repeatedly

If L = f(g(h(x))), the derivative w.r.t. x is f'(g(h(x))) · g'(h(x)) · h'(x). The chain rule. Backpropagation is just this, applied across every connection in a neural network, in a smart order.

"In a smart order" means: compute the loss first (forward pass), then walk backward through the network, multiplying gradients as you go. Reverse-mode automatic differentiation.

Why reverse, not forward?

Forward mode: compute gradient w.r.t. one input variable — efficient when there's one input and many outputs. Reverse mode: compute gradient w.r.t. one output (the loss) — efficient when there's one output and many parameters. Networks have millions of parameters but only one loss → reverse mode is the obvious choice.

A tiny network. Step through forward pass, then backward pass. Watch which gradient flows where.

Key takeaways

  • Backprop is the chain rule plus bookkeeping. It's exact (no approximation), and it's why training a 70B-parameter LLM is even feasible.
  • Frameworks (PyTorch, JAX, TF) build a computation graph on the forward pass, then walk it backward. You write the model; they handle the gradients.
  • "Vanishing gradients": multiplying many small numbers (e.g., sigmoid derivatives < 0.25) shrinks to zero through deep nets. ReLU and residual connections were invented to fix this.
  • "Exploding gradients": the opposite. Fixed with clipping, normalization (BatchNorm, LayerNorm), and careful initialization.
