Cache Hierarchy

A CPU register hit is essentially free. A trip to main memory is a hundred times more expensive. A trip to disk is a million times more expensive. Most "performance" work is about staying in the fast tiers.

The hierarchy

Each tier is dramatically smaller and dramatically faster than the next:

  • Register — < 1 ns — few hundred bytes
  • L1 cache — ~1 ns — 32–64 KB per core
  • L2 cache — ~4 ns — 256 KB – 1 MB per core
  • L3 cache — ~10 ns — 8–32 MB, shared across cores
  • Main memory (RAM) — ~100 ns — GBs
  • NVMe SSD — ~100 µs (100,000 ns) — TBs
  • Network round trip — ~50 ms (50,000,000 ns) — internet

Key takeaways

  • The hierarchy spans seven orders of magnitude in latency. Performance is mostly about which tier your data sits in.
  • Cache lines are 64 bytes — that's the unit of transfer. Accessing one byte loads the surrounding 63 too. Plan for it (next lesson: locality).
  • Modern CPUs prefetch aggressively: linear scans run fast because the prefetcher pulls the next line before you ask. Random access defeats it.
  • Once data is in network territory, software-level caching (CDN, Redis, app cache) takes over. The whole stack is just hierarchies inside hierarchies.
