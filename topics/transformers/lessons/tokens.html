Tokens & Embeddings

A neural network can't read text. The first job of any LLM pipeline is turning words into numbers — first by chopping text into tokens, then by mapping each token to a dense vector that captures meaning.

Tokenization

Not characters (too granular), not words (too many — including misspellings, names, languages). Modern LLMs use byte-pair encoding (BPE) or similar: a fixed vocabulary (~50k–200k tokens) of common substrings. "fundamentalofx" might be one token; "obscure_word" might be three.

Embeddings

Each token in the vocabulary has a learned vector — typically 512 to 12,288 dimensions. Tokens that mean similar things end up close in this vector space; differences encode relationships ("king" − "man" + "woman" ≈ "queen" is the famous example).

The embedding table is the first parameter of the model (vocabulary × embedding-dim). Pretty much every word the model knows starts as a row in that table.

Try it

Type text. See it chopped into tokens. Hover any token to see its (simulated) embedding vector.

Key takeaways

  • Tokens, not words. GPT-style tokenizers split "playing" into "play" + "ing"; "internationalization" might be 4 tokens.
  • Spaces are usually part of the following token. " Hello" and "Hello" can be different tokens.
  • Context length (4k, 32k, 1M) is measured in tokens, not characters. Roughly 1 token ≈ 4 characters of English.
  • Embeddings are the model's "first impression" of a token. They get refined by every subsequent layer.
