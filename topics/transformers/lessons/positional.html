Positional Encoding

Attention is a set operation — it doesn't naturally know "first," "second," "third." But word order is crucial: dog bites man and man bites dog are different things. So we mix position information into the embeddings before attention sees them.

The classic recipe (sinusoidal)

The original transformer used fixed sinusoids: PE(pos, 2i) = sin(pos / 10000^(2i/d)), PE(pos, 2i+1) = cos(pos / 10000^(2i/d)). Each position gets a unique 768-dim vector; nearby positions have similar vectors; the pattern is deterministic.

Modern variants

  • Learned absolute positions (GPT-2): the model learns its own position embeddings up to a max length.
  • RoPE (Rotary) (Llama, GPT-NeoX): instead of adding, rotate Q and K vectors based on position. Captures relative position naturally and extrapolates better.
  • ALiBi: bias attention scores by distance. No explicit position embeddings at all.

All are answering the same question: how do we inject "what position is this token at?" into a model that's otherwise permutation-invariant.

Try it

See the sinusoidal pattern. Hover positions to see their vectors. The pattern is structured — nearby positions overlap, distant ones don't.

Key takeaways

  • Without positional info, "the cat sat on the mat" and any permutation of those words would look identical to attention. Order matters.
  • Position is encoded once, at the input. Every subsequent layer inherits it via the residual stream — no need to re-inject.
  • "Context length extension" research (RoPE scaling, position interpolation) is all about making position encodings work beyond what the model was trained on.
  • For images, the same idea uses 2D positional embeddings; for audio, 1D again but on time steps.
