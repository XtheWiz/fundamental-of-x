The Transformer Block

One transformer block: attention plus a feed-forward network, with residuals and layer norm wrapped around both. Stack 32 of these and you have Llama 7B; 96 and you have GPT-3. The block is the unit; the network is the repetition.

The data flow

  1. Input: a sequence of embeddings.
  2. LayerNorm → normalize each token's vector.
  3. Multi-head attention → tokens mix information across positions.
  4. Residual connection: add the attention output back to the input.
  5. LayerNorm again.
  6. Feed-forward network: a two-layer MLP applied independently to each token (no mixing).
  7. Residual connection: add the FFN output back.

That's one block. Output goes into the next block's input. Repeat N times.

What each part does

  • Attention: information sharing across positions. The horizontal layer.
  • FFN: per-token compute, expanding then contracting the representation. The vertical layer. Believed to store much of the model's "facts."
  • Residuals: allow the original signal to pass through unchanged if needed. Critical for training deep stacks.
  • LayerNorm: keeps activations from drifting in scale. Stabilizes training.

Key takeaways

  • The block is intentionally repeatable. The same shape stacks deep — that's what makes scaling cheap to engineer (more compute, more layers, same code).
  • "Pre-norm" (LayerNorm before each sub-block, as shown) is the modern default; "post-norm" was the original 2017 design. Pre-norm trains more stably.
  • The FFN typically expands dimension 4× (e.g., 768 → 3072 → 768) — that "hidden expansion" is where much capacity lives.
  • About 30–50% of a model's parameters live in the FFNs; another 30–40% in attention; the rest in embeddings + the output head.
